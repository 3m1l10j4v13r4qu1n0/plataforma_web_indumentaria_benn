# Definition of Ready (DoR)
Una Historia de Usuario no puede moverse a la columna "En Desarrollo" en Trello si no cumple:
- [ ] Tiene el formato estándar (Como... quiero... para...).
- [ ] Tiene Criterios de Aceptación en formato Gherkin (Dado/Cuando/Entonces).
- [ ] Tiene al menos 2 ejemplos en Specification by Example.
- [ ] Tiene definidos los casos de prueba TDD (positivo y negativo/borde).
- [ ] Backend y Frontend han revisado los archivos `.md` específicos de la HU y no tienen dudas bloqueantes.
