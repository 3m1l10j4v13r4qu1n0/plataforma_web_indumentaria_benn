Historia de Usuario N°4
**Como** cajero,
**quiero** pedir el ticket de compra,
**para** validar que la venta exista en el sistema.

Tarjeta
El ticket será obligatorio para realizar cambios. Por política de negocio, no se deben permitir cambios sin comprobante.

Criterios de Aceptación
- Dado que un cliente solicita un cambio, cuando el cajero inicia el proceso, entonces el sistema debe solicitar el ticket de compra.
- Dado que el cliente no presenta comprobante, cuando se intenta continuar, entonces el sistema no debe permitir avanzar.
- Dado que se ingresa un número de ticket, cuando se valida, entonces el sistema debe verificar que el ticket exista en el sistema.

Specification by Example
- Ticket válido = cambio permitido
- Ticket inexistente = cambio rechazado

Acceptance TDD
- Buscar ticket existente
- Intentar cambio sin ticket y validar mensajes de error
