Historia de Usuario N°7
**Como** cajero,
**quiero** generar un ticket al finalizar la venta,
**para** entregar un comprobante al cliente.

Tarjeta
Cada venta debe registrar automáticamente un comprobante. La empresa necesita mantener registro de todas las operaciones y el ticket es necesario para futuros cambios.

Criterios de Aceptación
- Dado que el cajero ha finalizado una venta, cuando confirma la operación, entonces el sistema debe generar un ticket único con número de comprobante.
- Dado que se genera un ticket, cuando se procesa la venta, entonces el sistema debe guardar la fecha y los productos vendidos en el registro.
- Dado que ocurre un error de impresión, cuando el sistema intenta generar el comprobante, entonces debe mostrar una advertencia al cajero.

Specification by Example
- Venta confirmada = ticket generado
- Error de impresión = mostrar advertencia

Acceptance TDD
- Generar ticket correctamente y verificar datos registrados (caso positivo)
- Validar manejo de error de impresión y número único de comprobante (caso negativo/borde)
