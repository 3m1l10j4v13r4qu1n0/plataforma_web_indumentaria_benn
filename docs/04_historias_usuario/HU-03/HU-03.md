Historia de Usuario N°3
**Como** vendedor,
**quiero** verificar el estado del producto devuelto,
**para** aceptar solamente productos en buenas condiciones.

Tarjeta
El sistema debe controlar que el producto esté sin uso y con etiqueta. La empresa solo acepta cambios de productos sin uso.

Criterios de Aceptación
- Dado que un cliente devuelve un producto, cuando el vendedor registra el estado, entonces el sistema debe validar que el producto conserve su etiqueta.
- Dado que el producto no presenta signos de uso, cuando se evalúa para cambio, entonces el sistema debe permitir el cambio.
- Dado que el producto está en mal estado o presenta uso, cuando el vendedor intenta procesar el cambio, entonces el sistema debe rechazar el producto.

Specification by Example
- Producto nuevo y etiquetado = cambio válido
- Producto usado = cambio rechazado

Acceptance TDD
- Validar estado del producto y existencia de etiqueta
- Probar rechazo de productos dañados
