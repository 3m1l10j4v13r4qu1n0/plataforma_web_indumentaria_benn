Historia de Usuario N°6
**Como** vendedor,
**quiero** consultar el stock actual de productos,
**para** informar correctamente a los clientes.

Tarjeta
Los vendedores necesitan ver el stock actualizado en tiempo real. Actualmente deben preguntar manualmente en depósito, lo que genera demoras en la atención.

Criterios de Aceptación
- Dado que el vendedor necesita consultar un producto, cuando ingresa el nombre o código en el sistema, entonces el sistema debe mostrar la cantidad disponible.
- Dado que el producto existe en el catálogo, cuando se ejecuta la búsqueda, entonces el sistema debe mostrar su stock actualizado en tiempo real.
- Dado que el producto no existe en el sistema, cuando se intenta buscar, entonces el sistema debe informar un mensaje de error indicando que no existe.

Specification by Example
- Producto existente = muestra cantidad
- Producto inexistente = mensaje de error

Acceptance TDD
- Buscar producto válido (caso positivo)
- Buscar producto inexistente (caso negativo/borde)
